# Книга о вкусной и здоровой пище — веб-приложение

865 рецептов — 634 из издания 1952 года и 231 из личного кулинарного
блокнота (с фотографиями), разложенные по 15 тематическим разделам,
с поиском и избранным. Работает как обычный сайт и ставится на Android
(и iPhone) одним касанием, как настоящее приложение — иконка на рабочем
столе, открывается без адресной строки, работает офлайн. Поддерживает
светлую/тёмную тему и добавление собственных рецептов (с фото).

## Как установить на телефон

Чтобы кнопка «Добавить на главный экран» появилась по-настоящему (с офлайн-
работой через service worker), файлы нужно разместить на HTTPS-хостинге —
просто открыть index.html с телефона как локальный файл для полноценной
установки не получится (браузер такое не разрешает).

Самый быстрый бесплатный способ — **GitHub Pages**:

1. Создайте новый репозиторий на github.com (можно приватный).
2. Загрузите туда все файлы и папки из `recipe-app` **сохраняя структуру**
   (включая папку `images/`) — перетащить через веб-интерфейс GitHub
   (Add file → Upload files) можно сразу всей папкой в современных браузерах.
3. Settings → Pages → Source → выберите ветку `main` → Save.
4. Через минуту сайт появится по адресу вида
   `https://ВАШ_ЛОГИН.github.io/НАЗВАНИЕ_РЕПОЗИТОРИЯ/`.
5. Откройте эту ссылку в Chrome на телефоне → меню (⋮) →
   **«Добавить на главный экран» / «Установить приложение»**.

Если уже ставили более раннюю версию приложения — переустанавливать не
нужно, просто перезалейте изменившиеся файлы поверх старых; при следующем
открытии приложение само обновится (версия кэша повышена).

Альтернативы: Netlify Drop (netlify.com/drop — просто перетащить папку,
без регистрации), Vercel, или любой обычный хостинг.

## Структура файлов

- `index.html` — разметка
- `style.css` — оформление (зелёная палитра, светлая/тёмная тема)
- `app.js` — вся логика: категории, поиск, избранное, свои рецепты, фото
- `data.js` — база рецептов (865 шт.)
- `images/notebook/` — фотографии рецептов из кулинарного блокнота
- `manifest.json`, `sw.js` — для установки и офлайн-работы
- `icon-*.png` — иконки

## Свои рецепты и фото (через интерфейс)

Кнопка **+** внизу справа открывает форму добавления рецепта. В форме
есть поле «Фото» — можно выбрать снимок с телефона, он покажется
превью-миниатюрой; крестиком поверх превью фото убирается. Такие
рецепты (и их фото) хранятся прямо в памяти телефона — в браузерном
localStorage, ничего никуда не загружается.

## Как добавить/удалить фото рецепта через код (для книжных/блокнотных)

Через интерфейс редактируются и получают фото только *свои* рецепты
(добавленные кнопкой +). Для рецептов из книги 1952 года или из
кулинарного блокнота фото можно добавить или убрать прямо в `data.js`
— это обычный текстовый JS-файл с массивом объектов:

```js
const RECIPES = [
  {"id":1,"t":"Название рецепта","c":"РАЗДЕЛ","s":"ПОДРАЗДЕЛ","p":43,
   "i":"строка ингредиентов","b":"текст рецепта",
   "img":"images/notebook/20012-baton-1.jpg"},
  ...
];
```

**Чтобы добавить фото** к любому рецепту:
1. Положите файл фотографии в папку `images/` (можно в подпапку, например
   `images/my/`), любой распространённый формат (jpg/png/webp).
2. Найдите нужный рецепт в `data.js` (проще всего — поиском по названию
   в текстовом редакторе, `t":"Название рецепта"`).
3. Добавьте (или замените) у этого объекта поле `"img"` — путь к файлу
   относительно `data.js`, например `"img":"images/my/photo1.jpg"`.

**Чтобы удалить фото** — удалите поле `"img"` целиком у этого рецепта
(или поставьте `"img":null`).

**Чтобы добавить фото внутрь текста рецепта** (между шагами, как в
рецептах из блокнота) — вставьте в поле `"b"` внутри текста строку вида
`[[IMG:images/my/photo2.jpg]]` на отдельной "строке" (`\n` перед и
после в JSON-тексте) — приложение само отрисует её как картинку в этом
месте текста.

После правки файла `data.js` просто перезалейте его на хостинг — фото
и правки появятся у всех, кто откроет/обновит приложение.

## О качестве данных

Рецепты 1952 года извлечены автоматическим распознаванием текста из
скана книги — у части рецептов в конце текста могли остаться небольшие
следы брака OCR; у каждого указана страница оригинала для сверки.
Рецепты из кулинарного блокнота перенесены из экспортированных HTML-
файлов вместе с фотографиями — качество текста и фото — как в
исходном экспорте.
